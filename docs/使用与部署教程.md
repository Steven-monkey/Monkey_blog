## 一、项目能做什么

| 能力 | 说明 |
|------|------|
| **学习笔记** | `content/notes/` 下的 Markdown → SQLite → `/notes` 展示 |
| **小项目** | `content/projects/` 下的 Markdown → `/projects` 展示 |
| **智慧箴言** | `/playground/wisdom`，LLM 生成箴言 + 生图 + 海报下载 |

| **阅读助手** | 文章页右下角 AI 聊天，SSE 流式对话 |
| **内容推荐** | 文章底部 "你可能也喜欢" |

## 三、写内容

### 3.1 笔记

在 `content/notes/` 创建 `.md` 文件：

```yaml
---
title: 我的笔记
date: 2026-05-06
tags: [Python, Docker]
summary: 一句话摘要
slug: my-note   # 可选，不写则用文件名
---
正文内容...
```

### 3.2 项目

在 `content/projects/` 创建 `.md`：

```yaml
---
title: 项目名称
date: 2026-05-06
tags: [工具, 开源]
repo: https://github.com/user/repo
demo: https://demo.example.com
summary: 一句话介绍
---
项目说明...
```

### 3.3 页面速查

| 地址 | 页面 |
|------|------|
| `/` | 首页 (Hero + 关于 + 笔记/项目预览) |

## 五、部署到服务器

### 5.1 全新服务器（无 Nginx）

服务器安装 Docker 后：

```bash
git clone <仓库地址> /opt/blog
cd /opt/blog
# 创建 .env 配置 API 密钥
docker compose up -d --build
# 访问 http://服务器IP
```

### 5.2 已有 Nginx 的服务器（如宝塔面板）

```bash
git clone <仓库地址> /www/blog
cd /www/blog
docker compose -f docker-compose.prod.yml up -d --build
```

然后在服务器 Nginx 中添加反代配置（参考 `deploy/nginx.prod.conf`）：

```nginx
location / {
    proxy_pass http://127.0.0.1:8080;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_buffering off;      # SSE 流式必须关闭
    proxy_read_timeout 3600s;
}
```

详细步骤见 `deploy/DEPLOY_GUIDE.md`。

### 5.3 更新代码

```bash
cd /www/blog
git pull
docker compose -f docker-compose.prod.yml up -d --build
```

## 八、常见问题

**Q：改了笔记网页没变？**
A：需要重启 web 容器/进程触发重新同步。

**Q：AI 搜索/助手不可用？**
A：检查 `.env` 中 `LLM_API_KEY` 是否配置；访问 `/api/ai/status` 查看状态。

**Q：健康检查？**
A：`GET /healthz` → `{"ok": true, "sqlite": true}`
