# 个人 AI 博客：使用说明与服务器部署教程

本文说明如何**日常使用**本博客（写笔记、改项目页、用 AI 功能），以及如何**部署到你的 Linux 云服务器**（以 Docker 为主，适用于阿里云 ECS 等 Ubuntu 环境）。

---

## 一、项目能做什么

| 能力 | 说明 |
|------|------|
| **学习笔记** | `content/notes/` 下的 Markdown，同步进 SQLite 后在 `/notes` 展示 |
| **小项目** | `content/projects/` 下的 Markdown，在 `/projects` 展示，可配仓库/演示链接 |
| **智慧箴言** | `/playground/wisdom`，可接 LLM + 可选生图 API |
| **AI 搜索** | `/ai/search`，自然语言搜笔记与项目（需配置 `LLM_API_KEY`） |
| **阅读助手 / 推荐** | 文章页相关 API（需 LLM） |

权威内容在 **Git 仓库里的 Markdown**；运行时用 **SQLite** 存渲染后的 HTML 与元数据。改完 Markdown 后需要**重启应用进程**，才会重新同步到数据库。

## 二、目录结构（你需要关心的部分）

```
个人博客/
├── blog/                 # Flask 应用代码、模板、静态资源
├── content/
│   ├── notes/            # 学习笔记（*.md）
│   └── projects/        # 项目介绍（*.md）
├── data/                 # 本地默认 SQLite（data/blog.db，可不入库）
├── docs/                 # 文档（本教程等）
├── deploy/
│   ├── nginx.conf        # Nginx 反代配置（Compose 挂载）
│   └── nginx.ssl.example.conf  # 宿主机 Nginx + HTTPS 示例（可选）
├── docker-compose.yml   # web + nginx（对外 80）
├── Dockerfile
├── requirements.txt
├── run.py                # 本地可选：python run.py
└── .env                  # 密钥与配置（勿提交到 Git）
```

## 三、如何使用博客

### 3.1 写一篇「笔记」

1. 在 `content/notes/` 新建 `我的笔记.md`（文件名默认即 URL 中的 **slug**：`/notes/我的笔记`，若文件名含中文请确保服务器与 Git 编码为 UTF-8）。
2. 文件开头写 **YAML Front Matter**（上下用 `---` 包住），例如：

```yaml
---
title: 我的第一篇笔记
date: 2026-05-02
tags:
  - Python
  - Docker
summary: 一句话摘要（可省略；未配置摘要时可用 AI 生成，见下文）
slug: my-first-note          # 可选；不写则用文件名（不含 .md）
---
```

3. `---` 下面写正文，支持常见 Markdown（标题、列表、代码块、链接等）。
4. **重启**正在运行的 Web 容器或进程后，访问 `/notes` 与 `/notes/my-first-note`（若写了 `slug` 则以 slug 为准）。

### 3.2 写一篇「项目」页

在 `content/projects/` 新建 Markdown，Front Matter 示例：

```yaml
---
title: 某某小工具
date: 2026-05-02
tags: [工具, 开源]
repo: https://github.com/你的用户名/仓库名    # 可选，列表页会显示「仓库」按钮
demo: https://demo.example.com               # 可选，「演示」按钮
summary: 项目一句话介绍
---
```

正文写项目说明、技术栈、截图说明等即可。

### 3.3 页面与导航（速查）

| 地址 | 含义 |
|------|------|
| `/` | 首页：最近笔记与项目摘要 |
| `/notes` | 笔记列表 |
| `/notes/<slug>` | 单篇笔记 |
| `/projects` | 项目列表 |
| `/projects/<slug>` | 单个项目 |
| `/playground/wisdom` | 智慧箴言（AI 箴言 + 配图相关） |
| `/ai/search` | AI 智能搜索页 |
| `/healthz` | 健康检查 JSON（含 `sqlite` 字段） |

### 3.4 AI 与箴言相关配置（`.env`）

在项目根目录创建或编辑 `.env`（**不要**把真实密钥提交到公开仓库）。常用项如下：

**大模型（箴言、AI 搜索、阅读助手、自动标签/摘要等共用）**

```env
LLM_API_KEY=你的密钥
# 可选：接口地址与模型（默认 DeepSeek 兼容）
# LLM_API_URL=https://api.deepseek.com/chat/completions
# LLM_MODEL=deepseek-chat
```

**可选：兼容 OpenAI 的生图（智慧箴言背景图）**

```env
IMAGE_API_KEY=
IMAGE_API_BASE=https://api.openai.com/v1
IMAGE_MODEL=
```

**本地 SQLite 路径（可选）**

- 不写时：默认使用仓库根目录下 `data/blog.db`。
- Docker Compose 已设置容器内 `SQLITE_PATH=/app/data/blog.db`，数据在卷里持久化。

### 3.5 修改内容后如何生效

当前逻辑是：**启动时**把 `content/` 下 Markdown 同步进 SQLite。因此：

- 使用 **Docker Compose**：改完文件后执行 `docker compose restart web`（或 `docker compose up -d --build` 在改代码时）。
- 使用 **`python run.py` / `python -m blog.main`**：保存 Markdown 后停止进程再重新启动。

## 四、本地运行（开发或自测）

### 方式 A：Docker + Nginx（推荐，与线上一致）

Compose 会启动两个服务：**`web`**（Flask，仅容器内暴露 8080）和 **`nginx`**（对外映射 **80**，把请求反代到 `web`）。SSE（AI 阅读助手）已在 `deploy/nginx.conf` 里关闭 `proxy_buffering`，避免流式卡住。

1. 安装 [Docker Desktop](https://www.docker.com/products/docker-desktop/)（Windows/macOS）或 Linux 上的 Docker Engine + Compose 插件。
2. 在项目根目录放好 `.env`（至少需要时配置 `LLM_API_KEY` 等）。
3. 执行：

```bash
docker compose up --build
```

4. 浏览器打开：**`http://localhost`**（默认 **80** 端口，无需写 `:8080`）。  
5. 结束运行：在终端按 `Ctrl+C`，或另开终端执行 `docker compose down`（数据卷默认保留；若需清空 SQLite 卷，使用 `docker compose down -v`，**会删除卷内数据库**）。

查看日志：`docker compose logs -f web`；Nginx 问题可看 `docker compose logs -f nginx`。

### 方式 B：本机 Python（不装 Docker）

1. Python 3.12+（与 Dockerfile 一致即可）。
2. 在项目根目录：

```bash
pip install -r requirements.txt
```

3. 可选：设置 `SQLITE_PATH` 指向你想存放的 `.db` 文件路径。  
4. 启动：

```bash
python run.py
```

或：

```bash
python -m blog.main
```

5. 默认监听 `http://127.0.0.1:8080`（可用环境变量 `HOST`、`PORT` 修改）。

## 五、部署到你的云服务器（Ubuntu + Docker）

以下假设：**Ubuntu 22.04/24.04 64 位**，已有一台公网 ECS，你会用 SSH，安全组可自行在控制台配置。

### 5.1 服务器上安装 Docker

SSH 登录服务器后（示例命令，按官方文档微调即可）：

```bash
sudo apt update
sudo apt install -y ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin
sudo usermod -aG docker $USER
# 重新登录 SSH 后 docker 命令可免 sudo
```

验证：

```bash
docker version
docker compose version
```

### 5.2 把代码放到服务器

任选其一：

- **Git**：在服务器上 `git clone` 你的仓库，以后用 `git pull` 更新。
- **scp/rsync**：从本机把项目目录同步到服务器某路径，例如 `/home/ubuntu/个人博客`。

下文以项目目录为 `/home/ubuntu/blog` 为例。

### 5.3 配置 `.env`

在服务器项目根目录创建 `.env`（与 `docker-compose.yml` 同级），写入 `LLM_API_KEY` 等，**权限建议收紧**：

```bash
chmod 600 .env
```

### 5.4 启动服务

```bash
cd /home/ubuntu/blog
docker compose up -d --build
```

- 首次会构建镜像并启动容器，SQLite 文件在 Docker 卷中，路径由 `SQLITE_PATH=/app/data/blog.db` 指定。
- 查看日志：`docker compose logs -f web`；Nginx：`docker compose logs -f nginx`

### 5.5 安全组与端口

- Compose 由 **Nginx 容器映射主机 80 → 容器 80**，Flask 不直接暴露到公网。
- 在 **阿里云 ECS 安全组** 中放行 **入方向 TCP 80**。
- 浏览器访问：`http://你的公网IP`（无需端口）。

若本机 80 已被占用，可编辑 `docker-compose.yml` 里 nginx 的 `ports`，例如改为 `"8080:80"`，则用 `http://IP:8080` 访问，并在安全组放行对应端口。

### 5.6 更新网站内容或代码

```bash
cd /home/ubuntu/blog
git pull                    # 若使用 Git
docker compose up -d --build
docker compose restart web  # 仅改 Markdown 时重启 web 即可触发重新同步；若只改了 deploy/nginx.conf 则 restart nginx
```

## 六、可选：域名与 HTTPS（简要）

当前栈里 **Nginx 已在 Compose 中**，对外是 **HTTP 80**。要对访客使用 **https://你的域名**，常见两种做法：

1. **在 Compose 的 Nginx 上直接加 TLS**（进阶）：为 `nginx` 服务挂载 Let’s Encrypt 证书与私钥，在 `deploy/nginx.conf` 中增加 `listen 443 ssl` 的 `server` 块，并把 `docker-compose.yml` 里 nginx 增加 `443:443` 映射；安全组放行 **443**（及证书续期用的 **80**）。
2. **宿主机 Nginx 终止 TLS**：宿主机监听 443，反代到本机 `127.0.0.1:8080`。此时需在 `docker-compose.yml` 的 `web` 下增加 `ports: ["127.0.0.1:8080:8080"]`，并参考仓库内 **`deploy/nginx.ssl.example.conf`** 修改域名与证书路径；安全组放行 **443**。

域名 DNS 需配置 **A 记录**指向服务器公网 IP。

## 八、常见问题

**Q：改了笔记网页没变？**  
A：需要**重启** `web` 容器或 Python 进程，才会重新执行 Markdown → SQLite 同步。

**Q：`.env` 在仓库里找不到？**  
A：`.env` 通常在 `.gitignore` 中，需在服务器或本机**手动创建**；可参考上文变量名。

**Q：AI 搜索/助手不可用？**  
A：检查 `LLM_API_KEY` 是否配置；浏览器访问 `/api/ai/status` 查看 `enabled` / `configured`。

**Q：健康检查地址？**  
A：`GET /healthz`，应返回 `{"ok": true, "sqlite": true}`。
